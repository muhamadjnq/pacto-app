# راهنمای ساخت اپلیکیشن اندروید Pacto (ویندوز)

این راهنما قدم‌به‌قدم می‌گوید چطور از فایل اپ، یک فایل APK واقعی بسازی و در بازار / مایکت / گوگل‌پلی منتشر کنی.

> صبور باش. بار اول کمی طول می‌کشد چون چند ابزار باید نصب شوند. ولی بعد از آن، هر بار ساختن APK فقط چند دقیقه است.

---

## فهرست
1. پیش‌نیازها (نصب نرم‌افزارها)
2. آماده‌سازی پروژه
3. آفلاین کردن اپ (مهم)
4. ساخت پروژه‌ی اندروید
5. ساخت APK تست و نصب روی گوشی
6. ساخت نسخه‌ی نهایی امضاشده برای فروشگاه
7. انتشار در بازار / مایکت / گوگل‌پلی
8. رفع اشکال‌های رایج

---

## ۱) پیش‌نیازها

سه نرم‌افزار باید نصب شوند. به‌ترتیب:

### الف) Node.js
- به سایت https://nodejs.org برو و نسخه‌ی **LTS** را دانلود و نصب کن (Next → Next → Install).
- برای تست، یک پنجره‌ی **Command Prompt** (در منوی استارت تایپ کن: cmd) باز کن و بزن:
  ```
  node -v
  ```
  اگر یک عدد نسخه مثل v20.x نشان داد، درست نصب شده.

### ب) JDK (جاوا)
- Android نیاز به جاوا دارد. ساده‌ترین راه: **JDK 17** را از https://adoptium.net دانلود و نصب کن.

### ج) Android Studio
- از https://developer.android.com/studio دانلود و نصب کن.
- بار اول که بازش می‌کنی، یک ویزارد (Setup Wizard) می‌آید؛ همه‌چیز را روی پیش‌فرض بگذار و بگذار **Android SDK** را خودش دانلود کند (چند گیگ است، اینترنت لازم دارد).
- بعد از نصب، یک بار Android Studio را ببند.

> نکته: ممکن است برای دانلود این‌ها به فیلترشکن نیاز داشته باشی.

---

## ۲) آماده‌سازی پروژه

- این پوشه (`pacto-android`) را در جای مناسبی از کامپیوترت بگذار، مثلاً در `C:\pacto-android`.
- داخل پوشه، نوار آدرس بالای پنجره‌ی فایل را کلیک کن، تایپ کن `cmd` و Enter بزن. یک ترمینال **دقیقاً داخل همین پوشه** باز می‌شود.
- این دستور را بزن تا ابزارهای پروژه نصب شوند:
  ```
  npm install
  ```
  (چند دقیقه طول می‌کشد. به اینترنت نیاز دارد.)

---

## ۳) آفلاین کردن اپ (این مرحله را رد نکن)

فایل اپ به‌صورت پیش‌فرض چند کتابخانه را از اینترنت می‌گیرد. برای اینکه اپ روی گوشی **بدون اینترنت و بدون فیلترشکن** کار کند، یک بار این دستور را اجرا کن:

```
node make-offline.js
```

این اسکریپت کتابخانه‌ها را دانلود کرده و داخل فایل اپ جاسازی می‌کند. بعد از این، اپ کاملاً مستقل است.

> اگر این مرحله خطای شبکه داد، فیلترشکن را روشن کن و دوباره `node make-offline.js` بزن.

---

## ۴) ساخت پروژه‌ی اندروید

این سه دستور را به‌ترتیب بزن:

```
npx cap add android
npx cap sync
```

حالا یک پوشه‌ی `android` ساخته شده که پروژه‌ی واقعی اندروید است.

> هر وقت بعداً فایل `www/index.html` را عوض کردی، کافی است دوباره `npx cap sync` بزنی تا تغییرات وارد پروژه شوند.

---

## ۵) ساخت APK تست و نصب روی گوشی

برای ساختن یک APK سریع جهت تست:

```
cd android
gradlew.bat assembleDebug
```

(بار اول طول می‌کشد چون اجزای لازم را دانلود می‌کند.)

فایل APK ساخته‌شده اینجاست:
```
android\app\build\outputs\apk\debug\app-debug.apk
```

این فایل را با کابل یا تلگرام به گوشی بفرست، روی گوشی بازش کن و نصب کن (باید «نصب از منابع نامشخص» را اجازه بدهی). تمام — اپ روی گوشی‌ات نصب شد! ✅

> برای انتشار در فروشگاه از این نسخه‌ی debug استفاده **نکن**؛ مرحله‌ی ۶ را برو.

---

## ۶) ساخت نسخه‌ی نهایی امضاشده (برای فروشگاه)

فروشگاه‌ها فقط نسخه‌ی **امضاشده** را قبول می‌کنند. یک بار باید یک «کلید امضا» بسازی:

### الف) ساخت کلید امضا (keystore)
در ترمینال (داخل پوشه‌ی `android`) بزن:
```
keytool -genkey -v -keystore pacto-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias pacto
```
چند سؤال می‌پرسد (نام، شهر و…)؛ جواب بده و یک **رمز** انتخاب کن. **این فایل `pacto-key.jks` و رمزش را خیلی مراقب باش و جایی امن نگه دار** — برای هر بروزرسانی بعدی اپ به همین کلید نیاز داری.

### ب) معرفی کلید به پروژه
فایل `android\app\build.gradle` را با Notepad باز کن. داخل بخش `android { }` این را اضافه کن:
```gradle
    signingConfigs {
        release {
            storeFile file('pacto-key.jks')
            storePassword 'РМЗ_تو'
            keyAlias 'pacto'
            keyPassword 'РМЗ_تو'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled false
        }
    }
```
(به‌جای `РМЗ_تو` همان رمزی که انتخاب کردی را بگذار.)

### ج) ساخت فایل نهایی
برای **بازار و مایکت** (فرمت APK):
```
gradlew.bat assembleRelease
```
خروجی: `android\app\build\outputs\apk\release\app-release.apk`

برای **گوگل‌پلی** (فرمت AAB):
```
gradlew.bat bundleRelease
```
خروجی: `android\app\build\outputs\bundle\release\app-release.aab`

## ۷) انتشار در فروشگاه

### بازار (cafebazaar.ir)
1. به https://pishkhan.cafebazaar.ir برو و حساب توسعه‌دهنده بساز (رایگان).
2. «افزودن برنامه جدید» → فایل `app-release.apk` را آپلود کن.
3. آیکون، تصاویر، توضیحات و دسته‌بندی را پر کن و برای بازبینی بفرست.

### مایکت (myket.ir)
- مشابه بازار: حساب توسعه‌دهنده بساز و همان `app-release.apk` را آپلود کن.

### گوگل‌پلی
1. به https://play.google.com/console برو و حساب توسعه‌دهنده بساز (**هزینه‌ی یک‌بار ۲۵ دلار**، نیاز به کارت ارزی).
2. اپ جدید بساز و فایل **`app-release.aab`** را آپلود کن.
3. فرم‌های سیاست حریم خصوصی، رده‌بندی سنی و… را پر کن و بفرست.

> توجه: گوگل‌پلی به اپ‌هایی که فقط یک سایت داخل WebView هستند گاهی سخت‌گیری می‌کند. اگر رد شد، نگران نباش؛ بازار و مایکت بدون مشکل قبول می‌کنند، و برای گوگل‌پلی می‌توانیم بعداً نسخه‌ی نیتیو (Flutter) بسازیم.

---

## ۸) رفع اشکال‌های رایج

- **`npm` یا `node` شناخته نمی‌شود:** Node.js درست نصب نشده یا باید پنجره‌ی cmd را ببندی و دوباره باز کنی.
- **`gradlew.bat` کار نمی‌کند:** مطمئن شو با دستور `cd android` وارد پوشه‌ی android شده‌ای.
- **خطای SDK location not found:** یک بار پوشه‌ی پروژه را در خود Android Studio باز کن (Open → پوشه‌ی android را انتخاب کن) تا مسیر SDK را تنظیم کند، بعد دوباره امتحان کن.
- **صفحه‌ی سفید در اپ نصب‌شده:** یعنی مرحله‌ی ۳ (آفلاین‌کردن) را انجام نداده‌ای. `node make-offline.js` و بعد `npx cap sync` را بزن و دوباره بساز.
- **خطای جاوا/JDK:** مطمئن شو JDK 17 نصب است.

---

## خلاصه‌ی دستورها (برای مرور سریع)
```
npm install
node make-offline.js
npx cap add android
npx cap sync
cd android
gradlew.bat assembleDebug      (برای تست)
gradlew.bat assembleRelease    (APK برای بازار/مایکت)
gradlew.bat bundleRelease      (AAB برای گوگل‌پلی)
```

موفق باشی! اگر هر جای کار گیر کردی، عین پیام خطا را برام بفرست تا کمکت کنم.
